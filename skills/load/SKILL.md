## 執行步驟

1. **檢查進度檔案**：如果上方「已儲存的進度」顯示無紀錄，告知使用者並建議使用 `/save`，然後結束。

2. **快速總覽**：從 `progress.md` 解析並顯示：
   - **目標**：這次工作要達成什麼
   - **儲存時間**：上次儲存的時間
   - **5-Question Check**：快速驗證 context（如果有的話）

3. **進度狀態**：解析「進度狀態」表格，突出顯示：
   - 🔴 **未完成階段**：狀態為 `pending` 或 `in_progress` 的項目
   - ✅ 已完成階段：狀態為 `complete` 的項目（簡要列出）
   - 計算完成百分比（例如：3/5 完成 = 60%）

4. **待辦事項**：解析「待辦事項」區塊，列出所有 `- [ ]` 項目。

5. **關鍵 Context**：簡要呈現：
   - **決策紀錄**：之前做了哪些重要決定（避免重複討論）
   - **錯誤追蹤**：之前遇到哪些錯誤（避免重複踩坑）
   - **修改的檔案**：相關檔案列表（快速定位）

6. **比對當前狀態**：比較目前的 Git 狀態與進度檔案中記錄的「未提交的變更」和「最近 Commits」：
   - 如果有新的 commit → 提醒使用者
   - 如果檔案狀態有變化 → 列出差異
   - 如果一致 → 告知狀態與上次儲存時相同

7. **歷史紀錄摘要**：如果上方「歷史紀錄」不為空，簡要列出最近 3 筆歷史的儲存時間與工作摘要（僅一行摘要，不展開細節）。

8. **詢問使用者**：根據進度狀態和待辦事項，建議從哪裡繼續，並詢問使用者確認。

## 注意事項

- 使用**繁體中文**回覆。
- 重點是讓使用者快速回到上次的工作狀態，不需要逐字複述所有內容。
- **優先顯示未完成的項目**，已完成的可以簡略帶過。
- 如果進度檔案是舊格式（沒有新區塊），盡量解析並呈現可用的資訊，向下相容。
- 5-Question Check 是用來快速驗證你（AI）是否掌握足夠 context，不需要逐項唸給使用者聽。
